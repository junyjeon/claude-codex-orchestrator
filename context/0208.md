## 근거

### CLAUDE.md에 보안 주의사항을 상세히 기록

이 프로젝트는 외부 프로세스(Codex CLI)를 spawn하는 MCP 서버다. security.ts의 경로 검증, stdin pipe 패턴, 에러 sanitization이 보안의 핵심이다. AI가 이 파일들을 수정할 때 보안 함의를 인지해야 하므로 주의사항 섹션에 구체적으로 명시했다.

### docs/ 전면 재작성

코드가 v0.2.0에서 v1.0.0으로 진화했다. 도구 1개에서 4개로, 파일 4개에서 12모듈로, 환경변수 2개에서 7개로 늘었다. 기존 문서는 generate_code 하나만 설명하고 있어서 현재 코드와 완전히 괴리되어 있었다.

### 문서작성원칙 적용

agentic-boilerplate/guides의 문서작성원칙을 참고했다. 구분선(`---`), bold 레이블(`**핵심:**`), 과도한 강조를 제거하고 서술형 중심으로 문체를 개선했다. 특히 설계.md는 206줄에서 123줄로 줄이면서 정보 밀도를 높였다.

### suggest_model 벤치마크 기반 재설계

기존 추천 로직은 직관적 규칙(UI->Codex, bug fix->Codex)에 기반했다. 실제 벤치마크를 확인하니 Opus 4.6이 SWE-Bench 80.8%, LFG 9.25/10으로 버그 수정/다파일 작업에서 앞섰고, Codex 5.3은 Terminal-Bench 77.3%, Aider ~88%로 단일 파일/터미널 작업에 강했다. 이에 맞춰 UI/bug fix/security를 Claude 추천으로, scripting/boilerplate를 Codex 추천으로 변경했다.
