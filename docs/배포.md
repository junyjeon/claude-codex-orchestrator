# 배포

## 사전 준비

### Codex CLI 설치 및 로그인

먼저 Codex CLI를 설치하고 인증한다.

```bash
# 브라우저 인증
codex login

# 또는 API 키로 로그인
printenv OPENAI_API_KEY | codex login --with-api-key
```

인증이 완료되면 `echo "Write hello world" | codex exec -` 명령이 코드를 생성해야 한다.

ChatGPT Plus, Pro, Business, Edu, Enterprise 플랜이 필요하다.

## 설치

### npm 패키지 (권장)

```bash
npm install -g @junyjeon/claude-codex-orchestrator
```

```bash
git clone https://github.com/junyjeon/claude-codex-orchestrator.git
cd claude-codex-orchestrator
npm install
npm run build
npm link
```

## Claude Code 연결

### 1. MCP 서버 등록

전역 설정 (`~/.claude.json`):

```json
{
  "mcpServers": {
    "codex": {
      "command": "claude-codex-orchestrator"
    }
  }
}
```

프로젝트별 설정 (권장):

```json
{
  "projects": {
    "/path/to/your/project": {
      "mcpServers": {
        "codex": {
          "command": "claude-codex-orchestrator"
        }
      }
    }
  }
}
```

npx로 전역 설치 없이:

```json
{
  "mcpServers": {
    "codex": {
      "command": "npx",
      "args": ["-y", "@junyjeon/claude-codex-orchestrator"]
    }
  }
}
```

로컬 개발인 경우 dist/index.js를 직접 지정한다:

```json
{
  "mcpServers": {
    "codex": {
      "command": "node",
      "args": ["<프로젝트 경로>/dist/index.js"]
    }
  }
}
```

### 2. 환경변수

`.env` 파일을 서버 디렉토리에 만든다.

```bash
# 로깅
LOG_LEVEL=info                        # debug, info, warn, error

# 타임아웃 (밀리초)
CODEX_TIMEOUT=30000                   # generate/review용 (1000-300000)
CODEX_EXECUTE_TIMEOUT=120000          # execute용 (5000-600000)

# 보안: 허용 작업 디렉토리 (콜론 구분)
CODEX_ALLOWED_DIRS=/home/user/projects:/tmp

# 보안: danger-full-access 샌드박스 허용
CODEX_ALLOW_DANGER_SANDBOX=false

# 보안: --full-auto 모드 허용
CODEX_ALLOW_FULL_AUTO=false

# 보안: 최대 동시 Codex 프로세스 (1-10)
CODEX_MAX_CONCURRENT=3
```

기본값이 안전하게 설정되어 있으므로 .env 없이도 작동한다. 필요한 항목만 오버라이드하면 된다.

### 3. 활성화 (필요한 경우)

일부 환경에서는 `~/.claude/settings.json`에서 활성화가 필요하다.

```json
{
  "enabledMcpjsonServers": ["codex"]
}
```

## 테스트

Claude Code에서 간단한 요청을 해본다:

```
"binary search 함수를 TypeScript로 구현해줘"
```

4개 도구가 정상 등록되면:
- codex_generate: 코드 생성
- codex_execute: 자율 실행
- codex_review: 코드 리뷰
- suggest_model: 모델 추천

## 문제 해결

### Codex CLI not found

```
Codex CLI not found. Install: npm install -g @openai/codex
```

`which codex`로 PATH를 확인하고 Claude Code를 재시작한다.

### Authentication failed

`codex login`을 실행하고 브라우저에서 인증한 뒤 Claude Code를 재시작한다.

### Timeout

`.env`에서 CODEX_TIMEOUT 또는 CODEX_EXECUTE_TIMEOUT을 늘린다. 네트워크 연결을 확인한다.

### Path outside allowed directories

CODEX_ALLOWED_DIRS에 작업 디렉토리를 추가한다:

```bash
CODEX_ALLOWED_DIRS=/home/user/projects:/home/user/work
```

### MCP 서버가 감지되지 않음

1. `~/.claude.json` 설정 확인
2. Claude Code 완전히 재시작
3. 수동 테스트: `npx @junyjeon/claude-codex-orchestrator`
