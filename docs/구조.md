## 핵심 파일

### src/index.ts

MCP 서버의 엔트리포인트다. dotenv로 환경변수를 로드하고, Zod 스키마로 설정을 검증한다. 무효한 값은 안전한 기본값으로 폴백한다.

검증 대상:
- LOG_LEVEL: debug, info, warn, error
- CODEX_TIMEOUT: 1000-300000ms (기본 30000)
- CODEX_EXECUTE_TIMEOUT: 5000-600000ms (기본 120000)
- CODEX_ALLOWED_DIRS: 콜론 구분 허용 경로
- CODEX_ALLOW_DANGER_SANDBOX: boolean (기본 false)
- CODEX_ALLOW_FULL_AUTO: boolean (기본 false)
- CODEX_MAX_CONCURRENT: 1-10 (기본 3)

### src/server.ts

MCP 서버의 핵심이다. McpServer + registerTool() 패턴(SDK 1.26.0)으로 4개 도구를 등록한다.

도구:
- codex_generate: 코드 생성
- codex_execute: 자율 실행
- codex_review: 코드 리뷰
- suggest_model: 모델 추천

각 도구는 Zod 스키마로 입력을 검증하고, 구조화된 응답(text + structuredContent)을 반환한다.

### src/security.ts

보안 모듈이다. 세 가지 책임을 갖는다.

validateWorkingDir()는 경로 안전성을 검증한다. resolve()로 경로를 정규화하고, realpathSync()로 심볼릭 링크를 따라간 뒤, 허용 목록과 경계 인식 비교를 한다. `/home/user`가 `/home/username`을 매칭하는 걸 방지한다.

sanitizeErrorOutput()는 에러 출력에서 민감 정보를 제거한다. 홈 경로, 이메일, 자격증명, API 키, GitHub 토큰 7가지 패턴을 감지한다.

ProcessSemaphore는 동시 Codex 프로세스 수를 제한한다. 큐 기반 카운팅 세마포어로 FIFO 순서를 보장한다.

### src/codex/client.ts

Codex CLI를 spawn으로 실행하는 래퍼다. 쉘을 거치지 않아 command injection이 불가능하다. stdin 파이프로 프롬프트를 전달하고, JSONL 모드(--json)로 구조화된 이벤트를 받는다.

안전장치:
- 세마포어로 동시 실행 제한
- 1MB 출력 크기 제한
- SIGTERM → 5초 대기 → SIGKILL 캐스케이드
- 이중 resolve 방지

### src/codex/parser.ts

Codex CLI의 JSONL 출력을 파싱한다. 줄 단위로 JSON을 파싱하고, item.completed 이벤트에서 최종 메시지, 변경된 파일, 실행된 명령어를 추출한다.

### src/codex/prompts.ts

도구별 프롬프트 템플릿이다. generate, execute, review 각각에 맞는 프롬프트를 조립한다.

### src/router/suggest.ts

규칙 기반 모델 추천 엔진이다. API 호출 없이 즉시 응답한다. 작업 유형, 복잡도, 컨텍스트 크기, 키워드 매칭을 종합해서 claude 또는 codex를 추천한다.

신호 집계 방식이다. 각 요소가 가중치를 기여하고, 최종 점수 비율로 confidence를 계산한다.

### src/types/index.ts

모든 타입 정의다.

- CodexErrorCode: 8개 에러 코드 enum
- ApprovalMode, SandboxMode, ReviewFocus: Codex CLI 옵션 enum
- TaskType, Complexity: 라우터 입력 enum
- JSONL 이벤트 타입: ThreadStarted, TurnCompleted, ItemEvent, ErrorEvent
- 도구 입출력: GenerateInput/Output, ExecuteInput/Output, ReviewInput/Output, SuggestModelInput/Output
- 서버 설정: SecurityConfig, ServerConfig

## 설정 파일

### package.json

v1.0.0. ESM 모듈이다. `claude-codex-orchestrator` 바이너리를 제공한다.

의존성:
- @modelcontextprotocol/sdk 1.26.0
- zod 3.25.0
- dotenv 16.5.0

### biome.json

Biome 1.9.4로 린팅과 포매팅을 통합 관리한다. ESLint + Prettier를 대체한다.

### vite.config.ts

라이브러리 모드로 빌드한다. 엔트리는 src/index.ts, ESM 포맷, 미니파이 활성화. vite-plugin-dts로 타입 선언 파일을 생성한다.
