# 흐름

## 공통 플로우

모든 도구가 공유하는 실행 흐름이다.

```
사용자 요청
  ↓
Claude Code: 요청 분석 → 적절한 도구 선택
  ↓
MCP 서버: 입력 검증 (Zod 스키마)
  ↓
MCP 서버: 보안 검증 (경로 검증, 세마포어 획득)
  ↓
Codex CLI: spawn + stdin 파이프로 실행
  ↓
MCP 서버: JSONL 파싱 → 결과 구조화
  ↓
Claude Code: 결과 검증 → 사용자에게 제시
```

세마포어 획득은 동시 프로세스 제한(기본 3개)을 위한 것이다. 한도를 넘으면 FIFO 큐에서 대기한다.

## codex_generate 플로우

코드 생성 요청이다.

```
Claude Code → codex_generate 호출
  │
  │  task_description: "binary search 함수"
  │  language: "typescript"
  │  working_dir: "/project" (선택)
  │
  ▼
MCP 서버
  1. working_dir가 있으면 validateWorkingDir() 검증
  2. buildGeneratePrompt()로 프롬프트 조립
     → "Generate typescript code: binary search 함수"
  3. callCodex(prompt, {json: true}) 호출
  │
  ▼
Codex CLI
  - spawn('codex', ['exec', '-', '--json', ...])
  - stdin으로 프롬프트 수신
  - JSONL 이벤트 스트림으로 응답
  │
  ▼
MCP 서버
  4. JSONL 파싱 → 최종 메시지 추출
  5. 코드 블록과 설명 분리
  6. {code, explanation, language} 반환
```

## codex_execute 플로우

자율 실행 요청이다. 파일 수정, 명령어 실행을 포함한다.

```
Claude Code → codex_execute 호출
  │
  │  task_description: "테스트 추가"
  │  working_dir: "/project" (필수)
  │  approval_mode: "on-failure" (선택)
  │  sandbox: "workspace-write" (선택)
  │
  ▼
MCP 서버
  1. validateWorkingDir() 검증 (필수)
  2. fullAuto 결정:
     - approval_mode가 있으면 → 그대로 사용
     - 없고 CODEX_ALLOW_FULL_AUTO=true → --full-auto
     - 없고 false → on-failure 기본값
  3. buildExecutePrompt()로 프롬프트 조립
  4. callCodex(prompt, {json, sandbox, approvalMode, workingDir})
  │
  ▼
Codex CLI
  - do-run-inspect 루프 실행
  - 파일 생성/수정, 명령어 실행
  - JSONL로 각 단계 이벤트 출력
  │
  ▼
MCP 서버
  5. JSONL 파싱:
     - file_change 이벤트 → filesChanged 수집
     - command_execution 이벤트 → commandsRun 수집
     - agent_message → 최종 결과 추출
  6. {result, files_changed, commands_run, success} 반환
```

execute의 타임아웃은 CODEX_EXECUTE_TIMEOUT(기본 120초)을 사용한다. 다른 도구는 CODEX_TIMEOUT(기본 30초)이다.

## codex_review 플로우

코드 리뷰 요청이다.

```
Claude Code → codex_review 호출
  │
  │  code: "function foo() { ... }"
  │  review_focus: "security" (선택)
  │  file_path: "src/auth.ts" (선택)
  │  language: "typescript" (선택)
  │
  ▼
MCP 서버
  1. buildReviewPrompt()로 프롬프트 조립
     → review_focus에 따라 집중 영역 지정
     → JSON 포맷 응답 요청 포함
  2. callCodex(prompt, {json: true})
  │
  ▼
Codex CLI
  - 코드 분석 후 리뷰 결과 출력
  │
  ▼
MCP 서버
  3. 응답 파싱 (3단계 폴백):
     a. JSON.parse() 시도
     b. 실패 시 정규식으로 구조 추출
     c. 최종 실패 시 원문 텍스트 + 기본 구조
  4. {issues: [{severity, line, message, suggestion}], summary, score} 반환
```

## suggest_model 플로우

모델 추천이다. Codex CLI를 호출하지 않는다.

```
Claude Code → suggest_model 호출
  │
  │  task_description: "인증 시스템 설계"
  │  task_type: "architecture" (선택)
  │  complexity: "complex" (선택)
  │  context_size: 80000 (선택)
  │
  ▼
MCP 서버 (suggestModel 함수)
  1. 신호 집계:
     - task_type → 가중치 (architecture: claude 0.5)
     - complexity → 가중치 (complex: claude 0.4)
     - context_size → 50K/100K 임계값
     - 키워드 매칭 → 11개 패턴
  2. claude 점수 vs codex 점수 비교
  3. confidence = 승자점수 / 총점수
  4. {recommended, reasoning, confidence, alternative_scenarios} 반환
```

이 도구는 순수 JavaScript 로직이라 지연 없이 즉시 응답한다.

## 에러 처리 흐름

### Codex CLI 미설치

```
spawn → ENOENT 에러 → CODEX_NOT_FOUND 코드로 반환
→ Claude Code가 설치 안내 또는 자체 생성으로 폴백
```

### 인증 실패

```
Codex CLI → stderr에 인증 에러 → CODEX_AUTH_FAILED 코드로 반환
→ "codex login" 안내
```

### 타임아웃

```
타이머 만료 → SIGTERM 전송 → 5초 대기 → SIGKILL
→ CODEX_TIMEOUT 코드로 반환
→ Claude Code가 재시도 또는 자체 생성
```

### 출력 초과

```
stdout 수집 중 1MB 초과 → 프로세스 종료
→ CODEX_OUTPUT_TOO_LARGE 코드로 반환
```

### 경로 검증 실패

```
validateWorkingDir() → 허용 목록 외 경로 감지
→ "Path outside allowed directories" 에러 반환
→ CODEX_ALLOWED_DIRS 설정 안내
```

## 성능

일반적인 실행 시간:
- codex_generate: 2-10초 (함수 복잡도에 따라)
- codex_execute: 5-60초 (작업 범위에 따라)
- codex_review: 3-15초 (코드 크기에 따라)
- suggest_model: <1ms (규칙 로직, API 호출 없음)
